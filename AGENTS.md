# Agent operating contract (AITL)

> Generated by `aitl init agent`. This file is a **durable instruction** for any AI
> agent (Claude Code, Cursor, etc.) working in this repository. Treat the AITL MCP
> server **`aitl-js`** as the source of truth — not your in-context memory.

## Project scope

Use the project **`aitl-js`** for all MCP calls below.

## Before EVERY non-trivial decision — CONSULT THE MCP

Do not decide from memory. First query durable state via the `aitl-js` MCP server:

1. `search_memory` — semantic recall over prior memory/notes for this project.
2. `list_decisions` — the Architecture Decision Records already taken (don't contradict
   or re-litigate an accepted ADR; if you must, supersede it explicitly).
3. `get_repomap` — when the decision touches code structure.

If the MCP and your assumptions disagree, **the MCP wins** — or surface the conflict.

## After deciding / learning — PERSIST IT

1. `record_decision` — record any architectural decision as an ADR (Context / Decision /
   Consequences). Decisions that aren't written down don't exist.
2. `write_memory` — capture durable facts, preferences, and gotchas as structured memory
   (`type`: user | feedback | project | reference), keyed by a stable slug.
3. Record the prompt/instruction that drove the work (prompt history) so the session is
   reconstructable.

## Rules of thumb

- **Recall before reasoning.** A 1-second `search_memory` beats re-deriving context.
- **One decision → one ADR.** Keep them small and append-only.
- **Don't duplicate.** Update an existing memory/decision instead of creating a near-copy.
- **Scope everything** to project `aitl-js`.

## Quick reference (CLI equivalents)

```bash
aitl search "<query>" --project aitl-js          # semantic recall
aitl adr-sync --dir docs/adr --project aitl-js    # mirror ADRs into Mongo
aitl prompt list --project aitl-js                # prompt history
aitl ui --project aitl-js                          # browse/edit memories in the web UI
```
